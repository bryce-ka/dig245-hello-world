Files and naming conventions

One of the learn computing activities I completed was File organization and naming conventions, which taught me alot about digital naming conventions and why naming conventions are important.I learned to use descriptive names that you would be able to understand in the future when naming files and to also use version numbers and dates when saving different versions of files. The activity also taught me not to use spaces and to be attentive to capitalization when naming files. Another importnat consideration when building projects and establishing your file structure is to structure the files in a meaningful way that does more than just storing the files in the same place. Then naming conventions you use should also vary based on the language youre programming in. The standard naming convention for Python and R are both snake_case while Javascript and CSS use camelCase.

The Command Line

Another Learn Computing activity I completed was the command line. This tutorial taught me what the command line is and how to navigate my computers filesystem using the terminal/command line. I leaned that OS stands for operating system, CLI stands for command line interface, that bash stands for bourne again shell, and that programs ending in ".sh" are shell programs that can program and run cripts using the terminal. In addition to the terminology used when talking about the terminal, the activity also defined terms for actions in the terminal. For example, the command pwd prints the working directory of the terminal which is essentially the file path that the terminal is currenly on and the command whoami will return the the username of the person acessing the terminal. On my computer the whoami command returns "bryce" but it would return different things on others computers.
